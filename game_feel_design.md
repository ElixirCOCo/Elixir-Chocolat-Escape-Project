# 🎮 게임 감성 설계 문서 — game_feel_design.md
> 프로젝트: 쇼콜라의 칼퇴 대작전  
> 목적: 기능이 아닌 "감정"을 설계하는 문서  
> 작성: GPT (어드바이저) + Claude (기획팀장) + Gemini (기획감독)  
> ⚠️ 이 문서는 개발 중 감이 흐릴 때마다 다시 읽어라.

---

## 🧠 핵심 철학

> "플레이어는 최선의 선택을 하지 않는다. 후회할 선택을 한다. 그래서 게임이 재밌는 거다."

```
개발자 → 기능 완성
게임 디자이너 → 감정 조작   ← 우리가 가야 할 방향
```

**재미 루프의 진짜 정의:**
```
침투(스텔스) → 탐색 → 전투(발각) → 강화 → 리스크 선택 → 탈출 or 욕심 → 반복
                                              ↑
                                    이 순간이 게임의 전부다
```

---

## 🕵️ 스텔스 감성 설계

> 스텔스는 "긴장"이고, 발각은 "흥분"이다. 둘 다 재미있어야 한다.

### 스텔스 상태 연출
| 상황 | 연출 | 심리 효과 |
|------|------|----------|
| 적 시야 근처 | BGM 템포 느려짐 + 심장박동 약하게 | "들킬 것 같다" 긴장감 |
| 적 등 뒤 통과 | 화면 가장자리 살짝 붉어짐 | 아슬아슬함 |
| 스텔스 성공 탈출 | 조용한 승리음 + "퇴근 완료" 연출 | 성취감 극대화 |

### 발각 → 전투 전환 연출
```
발각 순간
→ 카메라 흔들림 0.3초
→ 경보음 + BGM 급격히 전투 음악으로 전환
→ 빌런 집결 시작
→ "라우드 모드" 돌입
```

## 💎 욕심 트리거 설계 (Greed Trigger System)

### 원칙: 아이템은 "좋아 보이는 것"이 아니라 "선택 압박 장치"다

### ✅ MVP 구현 세트 (1차 프로토타입용)

| 요소 | 내용 |
|------|------|
| 아이템 반짝임 + 색 등급 | 일반=흰색, 희귀=파랑, 전설=금색 파티클 |
| 게이트 반대편 배치 | 강한 아이템은 항상 출구 반대쪽 |
| 타이머 30% 이하 → 효과음 | 심장박동 사운드 시작 |
| 아이템 근처 접근 → 슬로우 | 0.9배속 (살짝만, 티 안 나게) |

### ✅ 본격 욕심 트리거 구조 (4차 폴리싱용)

**전설 아이템 등장 시 동시 발생:**
```
💎 전설 커피 머신 등장
→ "지금 돌아가면 안전함" 힌트 표시
→ 동시에: 타이머 30% 감소 + 빌런 스폰 속도 증가

결과: 먹으면 강해짐 + 상황 더 위험해짐
      이게 진짜 욕심 트리거다
```

**거짓 안전감 연출:**
```
탈출 게이트 근처 진입
→ 음악 잔잔해짐
→ 적 스폰 줄어듦
→ "아 이제 끝낼까...?" 느낌

근데 전설 아이템은 항상 게이트 반대쪽
→ "저거 하나만 먹으면 완벽한데..."
→ 이 순간이 게임이다
```

## 🧠 뇌를 건드리는 연출 (감각 조작)

> 사람은 논리로 안 움직인다. 착각으로 움직인다.

| 상황 | 연출 | 심리 효과 |
|------|------|----------|
| 강한 아이템 근처 | 사운드 변화 + 0.9배 슬로우 + 화면 밝아짐 | "이거 먹어야 될 것 같은데...?" |
| 탈출 게이트 근처 | 음악 잔잔 + 적 감소 | 거짓 안전감 → 욕심 유발 |
| 전설 아이템 등장 | 강렬한 파티클 + 등급 색 | 시각적 유혹 |

---

## ⏰ 타이머 단계별 연출 (공격적 버전)

| 단계 | 조건 | 연출 |
|------|------|------|
| 1단계 | 타이머 50% 이하 | HUD 색 변화 (흰색 → 주황) |
| 2단계 | 타이머 30% 이하 | 심장박동 사운드 시작 |
| 3단계 | 타이머 10% 이하 | 화면 가장자리 흔들림 + 숨소리 + 빨간 비네트 |

## 💬 HUD 문구 설계 원칙

> 안내하지 말고 부추겨라.

```
❌ 나쁜 예: "이거 먹으면 확실히 탈출 가능"  → 공지사항
❌ 나쁜 예: "위험합니다"                     → 경고문

✅ 좋은 예: "지금 돌아가면 안전함"           → 선택 압박
✅ 좋은 예: "근데 저건 놓치기 아까움"         → 욕심 자극
```

## 🎭 실패 연출 설계

> 실패도 재미있어야 계속 한다.

**실패 시 연출 흐름:**
```
1. 쇼콜라 피격 → 슬로우모션 0.5초
2. 화면 암전
3. 책상에 묶인 쇼콜라 (코믹 연출)
4. 빌런 "야근 추가요~" 대사
5. 서류 뭉치 책상에 쌓임
6. "조금만 더 하면 탈출할 수 있었는데..." 텍스트
7. 재시작 버튼 (리플레이 욕구 유도)
```

## 🎯 아이템 등급 체계

| 등급 | 색상 | 파티클 | 효과 강도 |
|------|------|--------|---------| 
| 일반 | 흰색 | 없음 | 약함 |
| 희귀 | 파랑 | 작은 파티클 | 중간 |
| 영웅 | 보라 | 중간 파티클 | 강함 |
| 전설 | 금색 | 강렬한 파티클 + 슬로우 | 매우 강함 + 리스크 |

## 🔧 폴리싱 효과 근거 (Tier 3 구현 시 참고)

| 기능 | 효과 | 이유 |
|------|------|------|
| 착지 카메라 출렁임 | 무게감·물리감 전달 | 중력 계산이 "느껴지는" 순간으로 변환, 캐릭터가 살아있다는 인상 |
| LineRenderer 탄환 궤적 | 사격 타격감 증폭 | Raycast를 시각화 — "맞았다"는 확신을 주는 저비용 연출 |
| ScriptableObject 1개 | 구조 확장성 증명 | 수치가 코드 밖에 있다는 것 자체가 데이터 주도 설계의 증거 |
| 피격 경직 0.1~0.2초 | HitStop과 시너지 | HitStop + 경직이 합쳐지면 타격감이 배가됨 |
| 착지 먼지 파티클 | 공간감·속도감 | 이동 속도가 빠를수록 착지 연출이 강조됨 |
| 발각 BGM 전환 | 긴장→흥분 전환 | 스텔스와 전투의 감성 분리를 청각으로 표현 |
