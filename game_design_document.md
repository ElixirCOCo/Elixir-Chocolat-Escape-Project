## 1. 게임 개요

### 1-1. 게임 제목
| 항목 | 내용 |
|------|------|
| 정식 제목 | 쇼콜라의 칼퇴 대작전 (가제) |
| 영문 제목 | Chocolat's Office Escape (가제) |

---

### 1-2. 기획 의도
"야근"은 현대 직장인이라면 누구나 공감하는 보편적 공포이자 유머 소재다.

이 게임은 VRChat 아바타 문화와 TPS 런앤건 장르를 결합하여,  
**"내가 공들여 꾸민 내 아바타가 총을 들고 야근에서 탈출하는"** 직접적이고 강렬한 판타지를 제공한다.

단순한 수업 프로젝트를 넘어, VRC 유저 커뮤니티를 핵심 타겟으로 한  
**"내 캐릭터로 즐기는 아케이드 탈출 액션"** 장르의 가능성을 탐색한다.

---

### 1-3. 타겟
| 구분 | 내용 |
|------|------|
| 핵심 타겟 | VRChat 유저 (자신만의 아바타를 보유한 유저) |
| 확장 타겟 | 게이머 전반 (10~30대) |
| 공통 정서 | 야근 공감, 직장인 유머, 아케이드 액션 재미 |

---

### 1-4. 테마
**"야근에서 탈출하라 — 오늘도 칼퇴는 사수한다"**

- 핵심 키워드: 탈출, 생존, 욕심, 반복
- 감성: 코믹 + 액션 반반
- 레퍼런스 정서: 더 디비전2의 긴장감 + 스트리노바의 캐릭터 매력 + 사무실이라는 친근한 배경

---

### 1-5. 배경
**무대: 대한민국 어딘가의 평범한 회사**

퇴근 시간은 진작 지났다.  
쇼콜라는 업무에 집중한 나머지 시계를 보지 못했고,  
동료들은 이미 다 퇴근한 상태다.

텅 빈 사무실에는 야간 근무자들이 남아 있다.  
그들의 목표는 단 하나 — 쇼콜라를 야근시키는 것.

---

### 1-6. 스토리 (시놉시스)
열심히 일한 죄로 혼자 남겨진 쇼콜라.  
퇴근하려는 순간, 야간 근무자들이 길을 막는다.

"박 대리님, 이것만 좀 봐줘요~"  
"오늘 야근 수당 드릴게요!"

쇼콜라는 사무실 곳곳을 뒤지며 사원증과 키카드를 찾아야 한다.  
빌런들의 방해를 뚫고, 제한 시간 안에 최종 게이트를 통과하면 탈출 성공.  
시간이 다 되거나 쓰러지면 — 야근 확정.

### 2-2. 장르
- **메인**: TPS (3인칭 슈터) · 숄더뷰
- **서브**: 스텔스 · 런앤건 · 로그라이트 요소

---

### 2-3. 아트 컨셉
| 항목 | 방향 |
|------|------|
| 캐릭터 | VRChat 아바타 스타일 (애니메이션풍, 개성 강함) |
| 배경 | 현실적인 사무실 — 파티션, 복도, 형광등, 책상 |
| 분위기 | 심야 사무실의 묘한 긴장감 + 코믹 연출 |
| 조명 | 형광등 깜빡임, 야간 분위기 |
| UI | 깔끔하고 직관적, 게임 내 분위기와 통일 |

---

### 2-4. 게임의 비전
> "VRC 유저라면 누구나 자신의 아바타로 플레이하고 싶어지는 게임"

- VRChat 아바타 문화와 게임을 연결하는 첫 번째 시도
- 1인 AI 협업 개발의 가능성을 증명하는 프로젝트
- 캐릭터를 "보는" 게 아니라 "조작하는 느낌"으로 설계

---

### 2-5. 레퍼런스 게임
| 게임 | 참고 요소 |
|------|----------|
| **더 디비전 2** | TPS 숄더뷰 조작감, 루트 시스템, 전투 긴장감 |
| **스트리노바** | 캐릭터 개성·매력, 아케이드 슈팅 템포 |
| **페이데이 2** | 스텔스→라우드 전환 구조, 긴장감 |
| **Vampire Survivors** | 런 내 강화 시스템, 중독성 루프 |
| **이스케이프 프롬 타르코프** | 익스트렉션 구조 (추후 모드 참고) |

---

### 2-6. 게임의 방향
- **수업용 (Track A)**: 핵심 재미 루프 — 스텔스·탐색·HitStop·탈출 판정
- **출시용 (Track B)**: 전체 시스템 — 야근 스택·보스·멀티·DLC·유저 모드

---

### 2-7. 게임의 지향점
- **손맛**: HitStop · 히트 피드백으로 타격감 극대화
- **선택의 재미**: 강한 아이템 vs 안전한 탈출 — 욕심 트리거
- **캐릭터 애착**: 내 아바타가 주인공 — 몰입감과 공유 욕구 자극
- **반복 플레이**: 야근 스택으로 누적 긴장감, 실패해도 재밌는 연출

---

### 2-8. 게임의 도전 요소
| 요소 | 내용 |
|------|------|
| 타이머 압박 | 제한 시간 내 탈출하지 못하면 패배 |
| 스텔스 vs 라우드 | 조용히 탈출할지, 싸우며 탈출할지 선택 |
| 야근 스택 | 반복할수록 난이도 상승 + 보스 해금 |
| 리스크 vs 보상 | 강한 아이템은 항상 출구 반대편 |
| 선택 기반 이벤트 | 랜덤 디버프 없음, 선택지로만 리스크 발생 |
| 억까 방지 | 체력 30% 이하 추가 디버프 금지 |

## 3. 주요 시스템

### 3-1. 핵심 재미 루프
```
침투 (스텔스 진입)
   ↓
탐색 (아이템 파밍)
   ↓
전투 (걸렸을 때 — 페이데이2 스타일)
   ↓
선택 (더 먹을까? 탈출할까?)
   ↓
탈출 판단 → 탈출 or 욕심 → 반복
```

### 3-2. 게임 모드

| 모드 | 내용 | 시기 |
|------|------|------|
| **칼퇴 대작전** (기본) | 스텔스 탈출 중심, 걸리면 전투 전환 | 1차 오픈 |
| **퇴사 전쟁** (익스트렉션) | 타르코프 스타일 전투 중심 탈출 | 추후 추가 |

---

### 3-3. 스텔스 시스템
- 기본 상태: 스텔스 (발소리·시야 범위 기반)
- 발각 시: 경보 → 빌런 집결 → 전투 모드 전환
- 전투 중 도주 성공 시: 다시 스텔스 복귀 가능
- 스텔스 클리어 보너스: 야근 스택 증가 속도 느림 (안전한 선택)
- 라우드 클리어 보너스: 야근 스택 증가 속도 빠름 (위험한 선택)

---

### 3-4. 야근 스택 시스템 (반복 플레이 핵심)
```
탈출 성공 → 야근 스택 +1
스택 증가 → 난이도 상승 + 특수 이벤트 활성화
스택 100% → 보스 스테이지 입장 가능 (선택)
```

- 스택 가속 선택 가능 (하이리스크·하이리턴)
- 보스 도전은 강제 아님 — 유저가 선택

---

### 3-5. 선택 기반 리스크 이벤트
> 랜덤 디버프 없음 — 모든 리스크는 선택으로 발생

스택 상승 시 선택지 제공:
```
[야근 옵션 선택]
A. 보상 증가 / 대신 적 강화
B. 아이템 확률 증가 / 대신 타이머 감소
C. 시야 감소 / 대신 공격력 증가
```

---

### 3-6. 억까 방지 시스템
| 규칙 | 내용 |
|------|------|
| 체력 30% 이하 | 추가 디버프 금지 |
| 연속 피격 | 일정 시간 보호막 |
| 타이머 임계치 | 강한 디버프 제한 |
| 강한 디버프 연속 | 연속 등장 금지 |

---

### 3-7. 디버프 설계 원칙
```
❌ 금지: 단순 능력치 감소 (속도↓, 데미지↓)
✅ 권장: 플레이 스타일 변화형 트레이드오프

예시:
반동 증가 → 대신 크리티컬 증가
시야 감소 → 대신 적 위치 표시
재장전 느림 → 대신 데미지 증가
```

---

### 3-8. 메타 보상 시스템
```
탈출 성공 → 재화 획득
재화 → 무기 해금 / 장비 구매 / 퍽 선택
```

- 밸런스를 건드리지 않는 **감성 보상** 별도 운영
  - 캐릭터 의상·컷신 해금·탈출 연출 변화·모션·무기 스킨

---

### 3-9. 캐릭터 시스템
| 항목 | 내용 |
|------|------|
| 초기 주인공 | 쇼콜라 (VRC 아바타 · 수업용) |
| 정식 출시 | 오리지널 캐릭터로 교체 예정 |
| 확장 | 다른 VRC 아바타 캐릭터 추가 (DLC) |
| 특화 시스템 | 스텔스 특화 / 전투 특화 / 해킹 특화 |
| 호감도 | 밸런스 영향 없는 감성 시스템 |

**캐릭터 교체 설계 원칙:**
- 모든 캐릭터는 `ICharacter` 인터페이스로 추상화
- `VisualRoot` 하위에만 아바타 배치 → 교체 시 Core 코드 무변경
- Humanoid 리깅 + Animation Rigging IK 보정 필수
- 정식 출시 전 교체 테스트 1회 필수 실행
